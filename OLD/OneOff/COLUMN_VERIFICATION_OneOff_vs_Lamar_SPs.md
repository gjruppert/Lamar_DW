**Reference:** OneOff scripts = `OLD/OneOff/*.sql`. Lamar SPs = `Lamar_Procedures/01_Common/`, `02_AP/`, `03_GL/`.

## Create_Tables_DDL.sql vs SPs

**Lamar_Index/Create_Tables_DDL.sql** was checked against the Lamar stored procedures. One mismatch was found and fixed:

- **D_GL_HEADER:** The DDL had column **GL_HEADER_ID**; all SPs (and OneOff) use **JE_HEADER_ID**. The DDL was updated to use **JE_HEADER_ID** so it matches the SPs.

All other tables in Create_Tables_DDL.sql match the column sets (and names) used by the corresponding SPs, including SCD2 columns where applicable.

---

## Conclusion

For all entities compared, **Lamar stored procedures output at least the same columns as the OneOff DDLs**. Dimension tables correctly add SCD2 columns (EFF_DATE, END_DATE, CRE_DATE, UDT_DATE, CURR_IND). One known bug in the OneOff balance script (LAST_UPDATE_DATE / LAST_UPDATED_BY swap) is fixed in the Lamar F_GL_BALANCES procedure. **Create_Tables_DDL.sql is now aligned with the SPs** after correcting D_GL_HEADER’s column name from GL_HEADER_ID to JE_HEADER_ID.
